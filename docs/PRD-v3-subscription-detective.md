# Invos Agent v3 — 隱形訂閱偵探 PRD

> 產品需求文件 v1.0 | 2026-03-21

---

## 產品願景

在「發票存摺」App 內嵌入 AI Agent 功能，自動從用戶的電子發票記錄中偵測「被遺忘的訂閱」，主動提醒用戶，幫助每年省下數千元不必要的訂閱費用。

---

## 目標用戶

台灣使用「發票存摺」App 管理電子發票的消費者。特別是：
- 訂閱了多個數位服務但記不清有哪些
- 試用期結束後不小心自動轉正式付費
- 想要了解自己每月固定支出有多少

---

## 核心場景

```
小美每月固定從信用卡扣款好幾筆訂閱，但她已經不記得自己到底訂了哪些服務。
有些試用期結束後自動轉正式付費，她根本沒注意到。
```

### AI Agent 對話範例

**AI 主動通知：**
> 「嗨小美，我發現你過去 3 個月每月都有 3 筆疑似訂閱扣款：
> Netflix NT$390、Spotify NT$149、還有一筆 NT$120 來自 Canva，
> 但你上次使用 Canva 的相關消費是 4 個月前了。
> 這筆 Canva 訂閱可能已經用不到，3 個月累計浪費了 NT$360。
> 要不要考慮取消？」

**用戶回覆：**
> 「天啊，我完全忘了 Canva！幫我列出所有疑似訂閱。」

**AI 回覆：**
> 「好的，根據你的發票記錄，你目前有 5 筆定期扣款，總計每月 NT$1,048。
> 其中 2 筆（Canva、某雲端空間）過去 3 個月沒有產生相關使用消費，
> 建議評估是否繼續。每年可省下約 NT$3,360。」

## 技術架構

### 資料來源
- 用戶的電子發票明細（CSV/API）
- 欄位：消費日期、消費品項、單價、個數、小計、店家名稱

### AI Agent 核心邏輯

#### 1. 訂閱偵測引擎

**偵測方法一：已知訂閱服務比對**
```
已知訂閱服務資料庫：
- YouTube Premium → Google Asia Pacific Pte Ltd
- ChatGPT Plus → OpenAI LLC / OpenAI OpCo, LLC
- Spotify → Spotify AB
- Netflix → Netflix International B.V.
- Uber One → 優食台灣股份有限公司（Uber One 訂閱費）
- Apple 訂閱 → Apple Distribution International
- Google Play → Google Asia Pacific Pte Ltd
- 電信月租 → 台灣大哥大、中華電信、遠傳電信
- 公用事業 → 台灣電力、自來水、瓦斯公司
```

**偵測方法二：模式識別（未知訂閱）**
```
條件：
1. 同一商家名稱
2. 相近金額（±5%）
3. 出現頻率：每月 1 次，連續 ≥ 2 個月
→ 標記為「疑似訂閱」
```

**偵測方法三：試用期轉正付費偵測**
```
條件：
1. 某商家首次出現小額或零元消費（試用）
2. 之後每月出現固定金額扣款
→ 標記為「試用轉正」，特別提醒
```

#### 2. 使用頻率分析

判斷訂閱是否「值得」：
```
- 該訂閱商家的「相關消費」頻率
  例：UberEats 訂閱 → 檢查外送訂單頻率
  例：Costco 會員 → 檢查好市多消費頻率
- 最後一次相關消費距今天數
- 若超過 60 天無相關消費 → 標記為「可能不需要」
```

#### 3. 省錢計算

```
每月訂閱總額 = Σ(所有偵測到的訂閱月費)
可省金額 = Σ(標記為「可能不需要」的訂閱月費)
年度浪費 = 可省金額 × 12
```

## Epic & Story 規劃

### EPIC-01: 訂閱偵測引擎

**STORY-01-01: 已知服務比對**
- 建立已知訂閱服務 → 商家名稱對應資料庫（≥30 個服務）
- 支援台灣常見數位訂閱、電信、公用事業
- 比對用戶發票中的商家名稱
- 輸出：確認的訂閱清單（服務名、金額、商家、頻率）

**STORY-01-02: 未知訂閱模式偵測**
- 掃描所有發票，找出符合「固定金額 + 固定週期 + 同商家」的消費
- 容許金額浮動 ±5%（稅金/匯率差異）
- 支援月繳/季繳/年繳
- 標記為「疑似訂閱」，請用戶確認

**STORY-01-03: 試用轉正偵測**
- 偵測某商家從零元/小額 → 固定金額的轉變
- 標記為「試用轉正」
- 計算試用結束後已扣款的累計金額

---

### EPIC-02: 使用頻率分析

**STORY-02-01: 關聯消費分析**
- 建立訂閱服務 → 關聯消費的對應表
  - Uber One → UberEats 外送訂單
  - Costco 會員 → 好市多消費
  - Foodpanda Pro → Foodpanda 訂單
- 分析用戶在訂閱期間的關聯消費頻率

**STORY-02-02: 閒置訂閱標記**
- 定義「閒置」：訂閱後 ≥60 天無關聯消費
- 計算閒置期間的累計浪費金額
- 依嚴重程度排序（浪費最多的排最前）

---

### EPIC-03: AI Agent 對話介面

**STORY-03-01: 主動偵測通知**
- AI 分析完成後，主動推送訊息給用戶
- 訊息格式：偵測到 N 筆訂閱、其中 M 筆可能不需要、可省 $X
- 語氣友善但直接，用具體數字

**STORY-03-02: 訂閱清單互動**
- 用戶說「列出所有訂閱」→ AI 回覆完整清單
- 清單包含：服務名、月費、商家、上次相關消費、狀態（使用中/可能閒置）
- 用 Markdown 表格呈現

**STORY-03-03: 追問與深入分析**
- 用戶可追問個別訂閱：「Canva 是什麼時候開始扣的？」
- AI 回覆：首次扣款日期、累計扣款金額、使用頻率分析
- 建議：保留/取消/降級

**STORY-03-04: 省錢報告**
- 用戶說「幫我算可以省多少」→ AI 產出完整報告
- 報告格式：
  ```
  📊 訂閱費用總覽
  ├── 每月總訂閱：NT$1,048（5 筆）
  ├── 建議保留：NT$688（3 筆）
  ├── 建議評估：NT$360（2 筆）
  └── 年度可省：NT$4,320
  ```

---

### EPIC-04: 首頁 AI 洞察卡片

**STORY-04-01: 訂閱偵測摘要卡片**
- 在「首頁」消費分析區塊下方插入 AI 洞察卡片
- 卡片內容：偵測到的訂閱數量、可能閒置數量、可省金額
- 點擊「查看詳情」進入 AI 聊天或訂閱管理頁

**STORY-04-02: 發票列表頁訂閱標記**
- 在「我的發票」列表中，訂閱類型的發票旁加標記：🔄
- 點擊訂閱類發票，顯示 AI 分析：「這是你的 YouTube Premium 月費」

---

### EPIC-05: 訂閱管理功能

**STORY-05-01: 訂閱管理頁面**
- 獨立頁面列出所有偵測到的訂閱
- 每筆顯示：服務名 / 月費 / 商家 / 狀態 / 最後使用 / 累計花費
- 支援標記為「已知，繼續保留」或「我要取消」

**STORY-05-02: 取消引導**
- 提供每個訂閱服務的取消方式
- 預建 20+ 常見服務的取消路徑
- 包含：App Store 管理 / Google Play 訂閱 / 官網取消連結 / 客服電話

**STORY-05-03: 月度訂閱報告推送**
- 每月初 AI 自動產出上月訂閱報告
- 包含：本月訂閱總額、新增訂閱、已取消、閒置警告
- 推播通知提醒用戶查看

---

### EPIC-06: 資料處理與部署

**STORY-06-01: 發票 CSV 資料處理**
- 解析發票存摺匯出的 CSV 格式
- 欄位對應：消費日期 / 消費品項 / 單價 / 個數 / 小計 / 店家名稱
- 日期格式：民國年 YYYMMDD → 西元轉換
- 處理折扣/優惠（負值小計）

**STORY-06-02: AI 模型整合**
- 使用 Gemini 2.5 Flash 作為對話 AI
- System Prompt 注入發票摘要 + 訂閱偵測結果
- 支援多輪對話上下文

**STORY-06-03: 部署上線**
- Next.js + Firebase Hosting（Cloud Run SSR）
- 獨立網址：invos-agent-v3.web.app 或同 Cloud Run URL
- Gemini API Key 透過環境變數注入
